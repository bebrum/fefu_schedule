# Расписание занятий: сбор, хранение и быстрые выборки

Небольшой pet-проект на Python: скрипт забирает расписание занятий, приводит ответ API к более удобному виду, сохраняет данные в SQLite и позволяет быстро посмотреть расписание на конкретный день и подгруппу.

Проект не претендует на большую production-систему. Это скорее практический пример, где я собрал данные из внешнего источника, немного их почистил, разложил по таблице и сделал понятные SQL-запросы для анализа.

## Что внутри

- `schedule_app.py` — основной скрипт для загрузки, обработки и просмотра расписания.
- `database.db` — SQLite-база с уже загруженными данными.
- `analysis_examples.sql` — примеры SQL-запросов для первичного анализа расписания.
- `request_config.example.json` — шаблон для cookies/headers, если нужно заново загрузить данные.
- `requirements.txt` — зависимости проекта.

## Что делает проект

- получает расписание из API ДВФУ;
- нормализует JSON-ответ: дата, время начала/конца, аудитория, тип занятия, преподаватель, подгруппа;
- сохраняет данные в SQLite;
- выводит расписание по дате и подгруппе;
- содержит SQL-запросы для простого анализа: нагрузка по типам занятий, самые загруженные дни, преподаватели, пары без аудитории.

В текущей базе: 1197 записей, 106 учебных дат, 14 уникальных дисциплин и 22 преподавателя. Период данных: с 2025-02-19 по 2025-06-28.

## Структура данных

Основная таблица — `schedule`.

| Поле | Описание |
| --- | --- |
| `title` | название дисциплины |
| `start_time` | время начала в минутах от начала дня |
| `end_time` | время окончания в минутах от начала дня |
| `date` | дата занятия в формате Unix timestamp |
| `classroom` | аудитория |
| `class_order` | номер пары |
| `class_type` | тип занятия |
| `subgroup` | номер подгруппы, если занятие не общее |
| `teacher` | преподаватель |

## Как запустить

Установить зависимости:

```bash
pip install -r requirements.txt
```

Посмотреть расписание из готовой базы:

```bash
python schedule_app.py --date 2025-02-19 --subgroup 1 --db database.db
```

Если дату не указать, скрипт возьмёт текущий день по часовому поясу Владивостока:

```bash
python schedule_app.py --subgroup 1
```

## Примеры аналитических запросов

Посчитать количество занятий по типу:

```sql
SELECT
    class_type,
    COUNT(*) AS lessons_count
FROM schedule
GROUP BY class_type
ORDER BY lessons_count DESC;
```

Найти самые загруженные учебные дни:

```sql
SELECT
    date(date, 'unixepoch', '+10 hours') AS lesson_date,
    COUNT(*) AS lessons_count
FROM schedule
GROUP BY lesson_date
ORDER BY lessons_count DESC
LIMIT 10;
```

Больше примеров лежит в `analysis_examples.sql`.

## Как обновить данные

В исходном варианте cookies и headers были прямо в коде. Для GitHub так лучше не делать, поэтому я вынес их в отдельный файл конфигурации.

1. Скопировать пример:

```bash
cp request_config.example.json request_config.json
```

2. Заполнить `request_config.json` актуальными cookies/headers из браузера.
3. Не коммитить `request_config.json` — он уже добавлен в `.gitignore`.

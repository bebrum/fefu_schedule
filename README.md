# Расписание занятий: сбор, хранение и быстрые выборки

Небольшой проект на Python: мотивация написать его появилась после постоянных сбоев в работе сайта с расписанием.

Проект не претендует на большую production-систему. Это скорее практический пример, где я собрал данные из внешнего источника, немного их почистил, разложил по таблице и сделал понятные SQL-запросы для анализа.

## Что внутри

- `schedule_app.py` — основной скрипт для загрузки, обработки и просмотра расписания.
- `database.db` — SQLite-база с уже загруженными данными.
- `analysis_examples.sql` — примеры SQL-запросов для первичного анализа расписания.
- `request_config.example.json` — шаблон для cookies/headers, если нужно заново загрузить данные.
- `requirements.txt` — зависимости проекта.

## Что делает проект

- получает расписание из API ДВФУ;
- нормализует JSON-ответ: дата, время начала/конца, аудитория, тип занятия, преподаватель, подгруппа;
- сохраняет данные в SQLite;
- выводит расписание по дате и подгруппе;
- содержит SQL-запросы для простого анализа: нагрузка по типам занятий, самые загруженные дни, преподаватели, пары без аудитории.

## Структура данных

Основная таблица — `schedule`.

| Поле | Описание |
| --- | --- |
| `title` | название дисциплины |
| `start_time` | время начала в минутах от начала дня |
| `end_time` | время окончания в минутах от начала дня |
| `date` | дата занятия в формате Unix timestamp |
| `classroom` | аудитория |
| `class_order` | номер пары |
| `class_type` | тип занятия |
| `subgroup` | номер подгруппы, если занятие не общее |
| `teacher` | преподаватель |
